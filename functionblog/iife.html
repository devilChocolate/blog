    立即执行函数IIFE

    wangkai 2022/1/10

    # 定义

    • 立即执行函数(IIFE Immediately Invoked Function Expression):声明一个函数,并马上调用这个函数就叫做立即执行函数

    # 注意事项

    # 一.立即执行函数是函数表达式，而非函数声明

      const fn1 = function () {
    return "这是函数表达式"
  }
  function fn2() {
    return "这是函数声明"
  }

    1. 函数表达式:函数在代码执行的到当前行的时候才被执行。如示例代码中的 fn1,当代码块执行到当前行,后面的函数才开始执行,fn1 才被赋值。f2 是函数声明,函数会被提前到代码最前面定义

    2. 以函数声明的方式定义的函数,函数名是必须的,而函数表达式的函数名是可选的。（函数声明整体会被提升到当前作用域的顶部，函数表达式也提升到顶部但是只有其变量名提升

    3. 以函数声明的方式定义的函数,函数可以在函数声明之前调用,而函数表达式的函数只能在声明之后调用。

    # 二.立即执行函数的语法

      //错误写法一
  function () {
    console.log(1);
  }()
  //SyntaxError: Function statements require a function name  函数语句需要名字
    • 因为 javaScrip 在解析时，遇到 function 关键字会默认为一个函数声明，而不是函数表达式，如果你不明确的告诉圆括号它是一个表达式，它会将其当作没有名字的函数声明并且抛出一个错误，因为函数声明需要一个名字。匿名函数属于函数表达式而不是函数声明
      //错误写法二
  function fn() {
    console.log(1);
  }()
  //SyntaxError: Unexpected token ')'   意外标记 ')'

    • 当圆括号放在一个函数表达式后面指明了这是一个被调用的函数，圆括号放在一个声明后面便意味着完全的和前面的函数声明分开了，此时圆括号只是一个简单的代表一个括号(用来控制运算优先的括号)

    • 正确写法 1: 用括号把整个函数定义和调用包裹起来

      (function () {
    console.log(1);
  }())
    • 正确写法 2: 用括号把函数定义包裹起来,后面再加括号
      (function () {
    console.log(1);
  })()
    • 立即执行函数一般也写成匿名函数
       (function () {
    console.log(2);
   }())

    # 三.防止 javaScript 解析出现误解,在立即执行函数表达式前加上分号

    • 解析器会认为"hellow"是一个函数，它将另一个函数作为其参数进行传递,如下
      const bar= "hellow"(function (){
    console.log(1);
  }())
    • 正确写法
      //正确写法
  const bar= "hellow"
  ;(function (){
    console.log(1);
  }())

    # 扩展

    • 除了在立即执行函数前添加分号，避免 JavaScript 引擎解析错误，还可以添加+ - ! ~ void 等一元运算符。因为+ - ! ~等运算符，都将函数声明转换成函数表达式，消除 javascript 引擎识别函数表达式和函数声明的歧义,告诉 javascript 引擎这是一个函数表达式，不是函数声明。可以在后面加括号，并立即执行函数的代码,其功能和使用()包裹相同。+是最安全的做法，因为- ! ~等运算符还会和函数的返回值进行运算,有时造成不必要的麻烦

    • 下面代码都可以运行

    +(function () {
  console.log(1);
})() -
  (function () {
    console.log(2);
  })();
!(function () {
  console.log(3);
})();
~(function () {
  console.log(4);
})();
void (function () {
  console.log(5);
})();

    # 作用

    • 一:隔离作用域(重要)

      ES6 之前，JavaScript 没有块级作用域。使用 IIFE 模拟块级作用域,避免全局污染。

      for(var i = 0 ; i < 5 ; i++){
    setTimeout(() => {
      console.log("循环中",i);
    }, 1000);
   }
  console.log("结果",i);

    结果如下:

    因为 for 循环中是一个块级作用域,var 关键字定义变量的只有全局作用域和函数作用域,所以 i 会提升到全局作用域。setTimeOut 是 Windows 对象中的方法,在指定毫秒数后执行回调方法,并且是一个异步过程(属于一个异步的宏任务),回调函数被存在异步队列中，等待执行栈清空后执行。for 循环结束完成后输出,i 的值为 5,执行栈清空。此时通 Event loop 从异步队列中获取回调压入栈中执行,在函数中未找到变量 i,于是沿着作用域链向上寻找到全局变量 i,所以依次输出 5 5 5 5 5 5

    使用 IIFE 模拟块级作用域:

      for(var i = 0 ; i < 5 ; i++){
    ((x)=>{
      setTimeout(() => {
        console.log(x);
      }, 1000);
    })(i)
  }
  console.log("结果",i);

    结果:

    每一次循环执行到立即执行函数都会创建一个属于自己的函数作用域,并且每一个 i 都通过参数传递并且保存在了函数作用域中。在 for 循环结束后,回调函数返回到定义时的环境,在函数作用域中就找到了变量 i,因此不需要沿作用域链查找到全局变量 i

    • 二:隐藏实现(常用于方法库的封装(如 jQuery),我们将需要提供的功能暴露出去,而自己的内部实现不希望被访问,通过 IIFE 隐藏起来)
      const window = {};
  (function () {
    let i = 1
    function add() {
      console.log("向外暴露的函数",++i);
    }
    function count() {
      console.log("我是被隐藏的函数");
    }
    window.$ = function () {
      return {add}
    }
  })()
  $().add()
