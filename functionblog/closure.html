    立即执行函数IIFE

    wangkai 2022/1/10

    # 闭包的概念

    • 闭包可以看成是持续保存在内存当中的函数作用域中的局部变量,我们可以使用Chrome浏览器查看闭包

    # 闭包的形成条件

    • 函数嵌套,并且内部函数使用了外部函数作用域中的变量

    • 内部函数被外部变量引用,保存在内存当中

      function A(params) {
    const name = "闭包中保存的变量"
    return function () {
      console.log(name);
    }
  }
  let scope = A()
  scope()

    # 闭包的创建时机

    • 外部函数被执行时，也就是内部函数被定义时形成

    # 闭包的销毁

    • 引用内部函数的变量不在引用内部函数
      function A(params) {
    const name = "闭包中保存的变量"
    return function () {
      console.log(name);
    }
  }
  let scope = A()
  scope()
  scope = null //引用内部函数的变量不在引用内部函数 闭包销毁 释放内存中保留的局部变量name

    # 闭包的作用

    • 在函数外部可以访问并操作函数内部的数据
    • 延长函数中局部变量的生命周期

    # 闭包的形成原理

    • 1. 正常情况下,函数内部可以沿着作用域链访问全局变量,而我们无法直接访问函数内部的局部变量,这是因为作用域的原因,作用域隔离了局部变量

    • 2. 函数作用域可以隔离局部变量更深层次的原因是因为在内存中,变量名和基本数据类型保存在栈内存中,对象类型的数据保存在堆内存中,在栈内存中保存了它的变量名和引用地址。函数运行结束后,释放了栈内存中的数据,所以保存在堆内存的数据没有指向它们的变量,成为了“垃圾对象”,在函数结束后的某个时刻被垃圾回收器回收。如下图

    函数创建时的变量分布在内存中

    函数结束时时的变量分布在内存中,此时栈内存被清空没有变量指向对象{name: '李四'}，成为垃圾对象被回收

    • 3. 函数也属于对象,但内部函数被return后被外部变量引用,所以没有被回收,一直保存在内存当中,入下图

    • 4. 内部函数被创建时(函数定义执行时),并且是嵌套在外部函数当中的,所以可以访问到外部的作用域对象(无法查看但存在的对象)。又因为内部函数使用了外部函数的数据,所以外部函数执行完成后,内部函数对外部函数的作用域对象还存在引用,因此外部函数的局部变量被保存在了内存当中
