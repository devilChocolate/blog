    # js数据类型

    • Number String Boolean undefined Null Object 以及ES6新增的Symbol和bigInt(是指安全存储、操作大整数,表示任意精度整数的基本数据类型)

    • 基本数据类型为Number String Boolean undefined Null,Symbol,bigInt

    • 引用类型为Object,常见的有Date Array Function

    • Symbol类型的值都是唯一的,并且Symbol类型的值可以作为对象的属性标识符使用,且无法被转化为字符串

    # 判断基本数据类型的方法

    • 对于基本数据类型,所以typeOf判断,例如 typeOf A,以小写字符串的形式返回A变量的类型

    • Null 类型虽然是基本数据类型,但是typeOf null 返回"object",这是javaScript的错误，原理是在JavaScript中二进制前三位都为0的话会被判断为 object类型,null的二进制表示是全0,自然前三位也是0,所以执行 typeof 时会返回"object"

    • undefined 和 Null两种类型值具有唯一性,undefined类型的值为undefined,Null类型的值为null,我们可以使用该特性判断undefined类型和Null类型

    • 虽然函数是一种特殊的对象,但是可以使用typeOf判断,返回“function”字符串

    • Boolean类型只有两个值,一个为true(真值)，一个为false(假值)。在js中,非假即为真,假值有以下值,Boolean值false,Number类型值0(包括0或者-0)和NaN,String类型'',undefined和null,其他的转化为Boolean都为true

    • NaN是一种特殊但Number类型（typeof NaN的结果为number）,表示不是一个合法的数字。NaN是唯一一个和自身不相等的值(NaN === NaN 为false),在js中使用isNaN判断是不是NaN,因为isNaN对非数字不起作用,它首先做的就是把这些值转换成数字，转换的结果可能为NaN，然后函数会错误地返回true

    # 判断引用数据类型的方法

    • 使用instanceof判断引用类型数据,格式为 a instanceof A a为实例对象,b为构造函数，返回true或者false,但是该方法不准确,例如数组类型array instanceof Object 也是true,原因是instanceof只是判断实例对象的原型链上是否出现构造函数的原型对象

    • function 可以使用typeOf判断

    • 使用Object.prototype.toString.call()判断引用类型数据
