    react中proxy配置

    wangkai 2022/3/10

    # 什么是跨域

    • 跨域指的是非同源策略请求。理解非同源策略请求首先要知道什么是同源策略请求。同源策略请求指协议/域名/端口号都一致,不满足同源策略请求就是非同源策略请求(如同boolean的非假即真原则)

    # 单一proxy如何配置

    • 在packge.json中追加如下配置:
       "proxy": "需要转发的服务器地址(例如： http://localhost:5001)"
    • 说明:
      1. 优点: 配置简单，前端请求资源时不需要添加任何前缀
      1. 缺点: 不能配置多个代理
      1. 工作方式: 上述方式配置代理，当请求了3000(前端本地端口为30000)不存在的资源时,那么请求会转发给5000(优先匹配前端资源)

    # 多个proxy如何配置

    1. 创建代理文件: 在src下创建配置文件: src/setupProxy.js。react脚手架内部会遍历该文件(文件名不能随意更改)

    2. setupProxy.js配置代理规则

      
  //setupProxy.js 使用 cjs配置方式

  const proxy = require('http-proxy-middleware') 
  module.exports = function()=>{
    app.use(
      proxy('/api1',{
        target: 'http://localhost:5001',
        changeOrigin: true,
        pathRewrite: {'^/api1': ''}
      }),
      proxy('/api2',{
        target: 'http://localhost:5002',
        changeOrigin: true,
        pathRewrite: {'^/api2': ''}
      }),
      proxy('/api3',{
        target: 'http://localhost:5003',
        changeOrigin: true,
        pathRewrite: {'^/api3': ''}
      })
    )
  }

    1. # setupProxy.js配置代理规则解析

    • # proxy第一个参数是路由的前缀名,任何加了该前缀的都会走后面配置的代理。如带有'/api1'前缀的请求都会转发给5001，'/api2'前缀的请求都会转发给5002。
    • # proxy第二个参数是proxy转发配置:
      1. target是配置转发的目标地址 (必写)
      1. changeOrigin: boolean类型值。控制服务器接受的请求头中的host字段的值。如果前端端口为3000，后端为5000时，若changeOrigin为true,服务器请求头中的host为localhost:5000;若changeOrigin为false,服务器请求头中的host为localhost:3000。也就是说如果该值为true,那么会将请求来源(前端)伪造成和后端同源的路径 (非必写)
      1. pathRewrite 去除前缀，保证交给后台服务器是正常的请求地址。例如后台接口地址为: http://localhost:5001/user/getCrad, 如果不重写路径,那么我们访问的实际上是http://localhost:5001/api1/user/getCrad。因为我们需要往路径中添加/api1前缀,使其可以走proxy代理,但添加完前缀又改变了路径，所以我们需要重写路径 (必写)
