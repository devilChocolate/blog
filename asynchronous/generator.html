    # generator函数区别于普通函数

    • 1. 写法上,Generator在函数在 function 后面，函数名之前有个 * 号
    • 2. 调用时,Generator函数在调用时不会立即执行,而是生成器。需要使用其中的next方法才能执行
    • 3. 通过yield关键字可以暂停执行流程
    • 4. Generator就是async/await语法糖的前身,使用async替代*号移动到function关键字前面,await替代yield关键字
      function * fn() {
    console.log(1);
    console.log(2);
    console.log(3);
  }
  fn() //不输出
-------------------------------------------------
  function * fn() {
    console.log(1);
    console.log(2);
    console.log(3);
  }
  const gen = fn() 
  gen.next() //输出 1 2 3
-------------------------------------------------
  function * fn() {
    yield console.log(1);
    yield console.log(2);
    yield console.log(3);
  }
  const gen = fn() 
  gen.next() //输出 1
  gen.next() //输出 2
  gen.next()  //输出 3
  gen.next() //无输出

    # generator函数的作用

    • 通过上面的例子我们可以看到,Generator函数配合yield关键字可以将函数的执行流挂起,可以作为解决异步的方案，也可以使代码交叉执行
      //我们可以等待yidld 后面执行完成 再继续执行(是不是和await很像)
  function * fn() {
    yield console.log(1);
    yield console.log(2);
    yield console.log(3);
  }
  const gen = fn() 
  gen.next()   //输出 1
  console.log("输出1结束"); //输出 输出1结束
  gen.next() //输出 2
  console.log("输出2结束"); //输出2结束
  gen.next() ////输出 3
  console.log("输出3结束"); //输出 1 //输出3结束
  gen.next() // Generator结束 无输出
  console.log("结束执行流"); //输出 结束执行流

    # generator函数每一次执行的输出

      function * fn() {
    yield "金角大王"
    yield "银角大王"
    yield "太上老君"
  }
  const gen = fn() 
  const n1 = gen.next()  // n1 = { value: 金角大王, done: false }

  const n2 = gen.next()  // n2 = { value: 银角大王, done: false }
  
  const n3 = gen.next()  // n3 = { value: 太上老君, done: false }

  const n4 = gen.next()  // n4 = { value: undefined, done: true }
  
  // next()执行会返回一个对象,其中value值是yield关键字后的结果,done是否结束
  // 第一次执行next  遇到yield停止 将后面的值金角大王返回,并且generator函数未执行完成,done为false
  // 第二次执行next  遇到yield停止 将后面的值银角大王返回,并且generator函数未执行完成,done为false
  // 第三次执行next  遇到yield停止 将后面的值太上老君返回,并且generator函数未执行完成,done为false
  // 第四次执行next  没有yield,所以返回undefined,done为true

--------------------------------------------------------------------------------------------------------
  //next 方法一般不带参数,如果带参数了,会作为上一个yield的返回值
  function * fn() {
    const name = yield "金角大王"
    console.log("上一个yield的返回值",name);
  }
  const gen = fn() 
  const n1 = gen.next("")   
  console.log("输出1结束",n1);
  const n2 = gen.next("孙悟空")    //作为上一个yield的返回值   因此name是孙悟空
