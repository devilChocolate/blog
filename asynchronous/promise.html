    # 关于Promise的理解

    • 1. 从语意上理解,Promise就是一个契约(承诺),在未来才会结束的事件完成后，才去执行对应的操作
    • 2. 从语法层面上看，Promise就是一个对象,可以获取异步操作的消息,提供了统一的API,对于不同的状态进行各自对应的操作
    • 3. 抽象一点,可以将Promise看成一个容器,里面保存了未来才结束的事件

    # Promise的特性

    • 1. 状态不受外界影响:Promise对象代表一个异步操作,有三种状态:pending(进行中),fulfilled(已成功)和rejectd(已失败)。只有异步操作的结果，可以决定前是哪一种状态,其他任何操作都无法改变这个状态

    • 2. 状态不可逆:一旦状态改变,就不会再变。Promise对象的状态改变只有两种可能: 从pending到fulfilled或者从pending到rejeced。当状态是fulfilled或者rejeced,状态就会凝固,一直保持这个结果，此时再使用resolve或者reject回调函数改变状态,也不会受影响

    # Promise对比callback

    • 1. 消除了造成回调地狱的隐患
    • 2. 不会产生控制反转，再拿到结果后我们改变状态，自己去获取结果，并且Promise的状态是不可逆的，即使多次调用也不会对其产生影响(安全可信任)

    # Promise的简单使用

    Promise 对象是一个构造函数,用来生成Promise实例。resolve函数:在异步操作成功时调用,并将异步结果作为参数传递出去,状态由pending -> fulfilled;reject函数:在异步操作失败时调用，也会把结果作为参数传递出去,状态由pending -> rejeced。then方法接受两个(可选)函数类型的参数，用于promise的状态改变后的处理

      new Promise((resolve,reject)=>{
    resolve('奔波儿灞')
  }).then(res=>{
    console.log("成功的promise",res);
  },reason=>{
    console.log("失败的promise",reason);
  })
  //结果: 成功的promise 奔波儿灞

      new Promise((resolve,reject)=>{
    reject('奔波儿灞')
  }).then(res=>{
    console.log("成功的promise",res);
  },reason=>{
    console.log("失败的promise",reason);
  })
  //结果: 失败的promise 奔波儿灞

    # Promise Api的方法

    # then 方法

    • 1. then 方法接受两个函数作为参数，第一个是接受resolve的传参，即成功时返回的参数。第二个是接受reject的传参，即失败时的传参（可选参数）

    • 2. then 方法前的代码是同步执行的，即使前面有setTimeout等定时器，也是顺序执行。定时器不是一个异步任务，而是会产生一个异步任务抛到异步的队列中等待， 这个任务执行的是定时器中的函数，而定时器本身是一个同步任务

    • 3. then 方法是异步的，只有改变Promise的状态时才加入到异步队列，剩余代码会继续执行

      new Promise((resolve,reject)=>{
    setTimeout(()=>{
     console.log("定时器")
    })
    console.log("继续执行resolve")
    resolve("成功时的返回”)
    console.log('继续执行剩余代码')
  }).then(res=>{
    console.log('接收成功的返回值')
  },reason=>{
   console.log('接收失败的返回值')
  })
  console.log('同步代码')
    • 4. Promise内部对异常进行了处理，如果发生异常也会加入到异步队列，但是并不是只有resolve或reject函数可以改变状态，如果发生异常也会改为rejeced状态，如下
      new Promise((resolve,reject)=>{
    try{
      //正常运行代码
    }catch{
      reject(throw Error('异常'))
    }
  }).then(res=>{
   //不执行
  },error=>{
   //捕获异常
  })
    • 5. then 方法then 方法中的两个函数参数，如果没有返回值，每一次处理会默认返回一个新的成功状态的Promise,值为undefined。当前resolve或reject状态的Promise,经过then方法中的参数函数处理都会返回一个新的成功的Promise,可以通过then方法一直链式调用
      //默认返回一个成功的promise
  const p = new Promise((resolve,reject)=>{
    reject("error")
  })
  .then(res=>{},reason=>{
    console.log("内部处理： ",reason);
    //此时默认返回一个成功的Promise 类似于reslove(undefined)
  })
  //当前的p 已经被处理了，并且状态是成功的
  p.then(res=>{
    console.log("成功",res); //输出 成功 undefined
  },()=>{
   //不执行
    console.log("失败");
  })
  .then(()=>{
    console.log("成功")
  },()=>{
    //不执行
    console.log("失败")
  })
    • 6. then 方法如果返回一个非Promise类型的且不具有then方法的对象(非鸭子类型)，则将其包裹在resolve的返回值
      const p = new Promise((resolve,reject)=>{
    reject("error")
  })
  .then(res=>{},reason=>{
    console.log("内部处理： ",reason);
    //此时将1作为resolve的返回值 类似于 resolve(1)
    return 1
  })
  //当前的p 已经被处理了，并且状态是成功的
  p.then(res=>{
    console.log("成功",res); //输出 成功 1
  },()=>{
    //不执行
    console.log("失败");
  })
    • 7. then 方法如果返回一个新的Promise对象，则下一次的then方法是对新对象的处理
      const p = new Promise((resolve,reject)=>{
    reject("error")
  })
  .then(res=>{},reason=>{
    return new Promise((resolve,reject)=>{
      reject("出现错误")
    })
  })
  //p 返回了一个新的promise对象 then是新的对象状态改变后的处理
  p.then(res=>{
    console.log("成功",res);
  },reason=>{
    //不执行
    console.log("失败",reason); //失败 出现错误
  })

    # finally 方法

    • 1. 不依赖状态，并且保证会执行
    • 2. 不接受参数
    • 3. 本质上是then方法的特例,finally 在then方法结束后调用,finally 方法的回调函数不接受任何参数，意味着没有办法知道前面的Promise的状态是fulfilled还是rejected。所以不管Promise对象最后的状态如何,都会执行finally中的操作,并且操作与状态无关,不依赖Promise的结果
      new Promise((resolve,reject)=>{
   resolve()
  })
  .then(res=>{
    console.log("成功时执行");
  },reason=>{
    console.log("失败时执行");
  })
  .finally(()=>{
    console.log("最后执行"); //无论是执行成功的函数resolve或者是失败的函数reject，都会执行
  })

    # catch 方法

    • 1. 方法在内部也是调用的Promise.prototyep.then方法中reject状态下的方法
    • 2. 用与发生错误时的回调函数,如果之前的错误被then函数处理，不会进入到catch函数中,因为then方法默认返回一个成功的Promise
      new Promise((resolve,reject)=>{
    throw Error("Promise 抛出的异常”)
  }).then(res=>{
    //不执行
  },error=>{
    //执行
    console.log("reject自动捕获"); //成功捕获到同步代码中抛出的异常
  }).catch(error=>{
    //不执行
    console.log("catch成功捕获");  //被then方法重的reject函数处理，所以不会输出
  })
    • 3. 若错误没有被处理,则会被catch捕获
      new Promise((resolve,reject)=>{
    reject("失败状态被catch捕获")
  })
  .then(res=>{
    //不执行
  })
  .catch(error=>{
    //执行
    console.log("因为没有reject处理，所以catch捕获失败状态"+error);
  })
    • 4. 如果Promise已经是成功状态了（调用了resolve）,那么即使再抛出错误也是无效的（状态不可逆）
      new Promise((resolve,reject)=>{
    resolve("Promise 成功")
    throw Error("Promise 失败")
  })
  .then(res=>{
    //执行
    console.log(res)
  })
  .catch(error=>{
    //不执行
    console.log(error);
  })
    • 5. 异步代码可以主动调用reject改变状态，但是无法主动捕获错误自动reject
      //同步中自动捕获
  new Promise((resolve,reject)=>{
    throw Error("Promise 抛出的异常")
  })
  .then(res=>{
    //不执行
  })
  .catch(error=>{
    //执行
    console.log("因为没有reject处理，所以catch捕获异常");
  })

      //异步中无法自动捕获
  new Promise((resolve,reject)=>{
    setTimeout(() => {
      throw Error("Promise 抛出的异常")
    }, 2000);
  }).then(res=>{
    //不执行
  }).catch(error=>{
    //不执行
  })

    # all 方法

    • 1. 将多个Promise实例包装成一个新的Promise实例，方法接受一个带Iterator迭代器的对象(Array Map Set)
      const p1 = new Promise((resolve,reject)=>{
    resolve("p1成功")
  })
  const p2 = new Promise((resolve,reject)=>{
    resolve("p2成功")
  })
  const p3 = new Promise((resolve,reject)=>{
    resolve("p3成功")
  })
  Promise.all([p1,p2,p3]).then(res=>{
    //执行
    console.log(res);
  },reason=>{
    //不执行
    console.log(reason)
  })
    • 2. 当参数中每一个Promise都是成功的时候返回一个结果数组,并且结果与参数一一对应
      const p1 = new Promise((resolve,reject)=>{
    resolve("p1成功")
  })
  const p2 = new Promise((resolve,reject)=>{
    resolve("p2成功")
  })
  const p3 = new Promise((resolve,reject)=>{
    resolve("p3成功")
  })
  Promise.all([p1,p2,p3]).then(res=>{
    //执行
    console.log(res);
  },reason=>{
    //不执行
    console.log(reason)
  })
    • 3. 当参数中的Promise失败时返回最先reject失败状态的值
      const p1 = new Promise((resolve,reject)=>{
    resolve("p1成功")
  })
  const p2 = new Promise((resolve,reject)=>{
    resolve("p2成功")
  })
  const p3 = new Promise((resolve,reject)=>{
    resolve("p3失败")
  })
  Promise.all([p1,p2,p3]).then(res=>{
    //不执行
    console.log(res);
  },reason=>{
    console.log(reason) //返回第一个失败的结果
  })
    • 4. 验证all方法返回第一个失败的结果，是指参数形式上的第一个(根据数组顺序)还是第一个reject状态的(执行结果)
      const p1 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      reject("p1失败")
    }, 3000);
  })
  const p2 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      reject("p2失败")
    }, 2000);
  })
  const p3 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      reject("p3失败")
    }, 1000);
  })
  Promise.all([p1,p2,p3]).then(res=>{
    //不执行
    console.log(res);
  },reason=>{
    console.log(reason);
  })

    上面的例子都会reject,但是第三个Promise最先被reject,所以返回了第三个Promise的结果。说明在all方法中，当发生错误时返回第一个被reject的Promise,是从被reject的顺序来衡量的，而不是它们在参数中的顺序。同时也说明了Promise.all方法是并行执行的，不需要等待前面的Promise完成，后面的Promise不依赖于前面Promise的结果

    # allSettled 方法

    • 1. 将多个Promise实例包装成一个新的Promise实例，方法接受一个带Iterator迭代器的对象(Array Map Set)
    • 2. 返回每一个Promise的处理结果数组，并且结果与参数一一对应，无论成功或失败
    • 3. 结果数组的每一项是一个对象，其中status代表Promise的状态，reason代表返回值
      const p1 = new Promise((resolve,reject)=>{
    resolve("p1成功")
  })
  const p2 = new Promise((resolve,reject)=>{
    reject("p2失败")
  })
  const p3 = new Promise((resolve,reject)=>{
    reject("p3成功")
  })
  Promise.allSettled([p1,p2,p3]).then(res=>{
    console.log(res);
  },reason=>{
    console.log(reason);
  })

    # race 方法

    • 1. 将多个Promise实例包装成一个新的Promise实例，方法接受一个带Iterator迭代器的对象(Array Map Set)
    • 2.返回最先改变状态的Promise,无论成功或失败,race是竞态关系
      const p1 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      resolve("p1成功")
    }, 3000);
  })
  const p2 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      reject("p2失败")
    }, 100);
  })
  const p3 = new Promise((resolve,reject)=>{
    setTimeout(() => {
      reject("p3成功")
    }, 300);
  })
  Promise.race([p1,p2,p3]).then(res=>{
    console.log(res);
  },reason=>{
    console.log(reason);
  })
    • 使用race对请求超时处理示例
      function ajax(url) {
    let xhr = new XMLHttpRequest();
    xhr.open("GET",url)
    xhr.send()
    xhr.onload = function(){
      if(this.status === "200"){
        resolve(this.response)
      }else{
        reject("axjx请求失败")
      }
    }
  }
 const p1 = ajax(url)
 const p2 = new Promise((resolve,reject)=>{
   setTimeout(() => {
      //如果ajax请求超过2000毫秒，就返回
      reject("请求超时")
