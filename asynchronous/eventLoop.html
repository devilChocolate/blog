    eventLoop 事件循环机制

    wangkai 2022/1/10

    # Event loop 中的角色:

    • Heap（堆）: 保存引用类型
    • Web APIS: 由宿主环境提供（如浏览器），可以执行定时任务，发送异步请求等操作（多线程)
    • Queue: 事件队列，FIFO 类型（first input, first output ）
    • Stack：执行栈, FILO 类型 ( first input, last output）

    # js 执行顺序:

    • JavaScript 是单线程语言。即代码从上往下依次执行，在遇到错误时终止
      function fn1() {
    fn2()
    console.log('a'); //语句1
  }
  function fn2() {
    fn3()
    console.log('B'); //语句2
  }
  function fn3() {
    console.log('C'); //语句3
  }
  fn1()
    • 顺序执行步骤： fn1入栈 -> fn2入栈 -> fn3入栈 -> 语句3入栈 -> 语句3出栈输出c -> 语句2入栈 -> 语句2出栈输出B -> fn2出栈 -> 语句1入栈 -> 语句1出栈输出A -> fn1出栈

    # 模仿event loop的执行过程(伪代码):

      const eventLoop = []
  while (true) { 
    if (eventLoop.length > 0) {
      const event = eventLoop.shift();
      try {
        event()
      }catch(err){
        reportError(err)
      }
    }
  }

    # Even loop 的工作流程:

    • 因为JavaScript是单线程执行的，所以在遇到异步的任务（如I/O读写，请求后端数据等），我们需要等待这些任务完成后才能继续执行代码。在等待这些任务时会堵塞页面的加载和渲染所以我们将这些任务交给我们的Web APIs执行，借助Web APIs是多线程的原理可以更高效的完成我们的任务。js单线程要做的不是发送请求等事件（或者说这不是主要的任务）,我们需要的是发送请求后拿到的结果。js执行的过程中将需要异步执行的任务抛出，然后继续执行同步的代码，等到同步代码执行完成后，再去异步队列获取任务结果。在异步完成后,会将结果放到queue事件队列中,我们需要通过 event loop将其加入到执行栈中执行。如下图:

    • 注意:就是setTimeout或者ajax请求,axios请求等,即使它们在几毫秒内完成,也是异步任务，需要在同步代码执行完成后才能被执行

    • 示例代码一

      setTimeout(() => {
    console.log('A');//A永远不会输出，因为同步任务是一个死循环永远不会结束，执行栈会一直被占用
  }, 0);
  while (true) {} //死循环
    • 示例代码二
      setTimeout(() => {
    console.log("A");
  }, 0);
  for (let index = 0; index < 100000000; index++) {
    console.log(index);
  }
    • 在代码示例2中,虽然定时器设置的时间是0毫秒,但是在0毫秒后不会输出A,因为此时for循环还未结束，执行栈一直被占用。
    • setTimeout设置的执行时间是最小执行时间,真实执行的时间大于等于 delay（延迟时间），我们设置的时间是最理想或者说至少需要延迟的时间
